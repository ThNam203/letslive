================================================================================
          VIDEO UPLOAD & TRANSCODE FEATURE - IMPLEMENTATION PLAN
================================================================================

CONTEXT
-------
Currently, VODs are only created when a livestream ends. The transcode service
captures the RTMP stream, transcodes via FFmpeg to HLS, uploads segments to
MinIO, and creates a VOD record. There is NO way for users to upload a
pre-recorded video like YouTube allows.

This plan adds: upload video -> save to storage -> queue for transcoding ->
FFmpeg produces HLS -> VOD is playable.

ARCHITECTURE OVERVIEW
---------------------

  Mobile/Web                         Backend
  ----------                         -------

  Upload video -----> Livestream Service
    (multipart)         |
                        +-> Save raw file to MinIO (raw-videos bucket)
                        +-> Create VOD record (status: "processing")
                        +-> Insert job into transcode_jobs table (PostgreSQL)

                      Transcode Service (new worker loop)
                        +-> Poll transcode_jobs for pending jobs
                        +-> Download raw file from MinIO
                        +-> FFmpeg transcode to HLS (reuse existing logic)
                        +-> Upload HLS segments to MinIO (reuse existing storage)
                        +-> Update VOD record (status: "ready", set playbackUrl)
                        +-> Delete raw file from MinIO (cleanup)

File: backend/livestream/migrations/0004_add_vod_upload_support.sql

  -- Add status field to vods table
  ALTER TABLE vods ADD COLUMN status VARCHAR(20) NOT NULL DEFAULT 'ready';
  -- 'ready' default preserves backward compat for existing livestream VODs
  -- Possible values: 'uploading', 'processing', 'ready', 'failed'

  ALTER TABLE vods ADD COLUMN original_file_url VARCHAR(2048);

  -- Transcode job queue (PostgreSQL as simple job queue)
  CREATE TABLE transcode_jobs (
      id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
      vod_id UUID NOT NULL REFERENCES vods(id) ON DELETE CASCADE,
      status VARCHAR(20) NOT NULL DEFAULT 'pending',
      attempts INT NOT NULL DEFAULT 0,
      max_attempts INT NOT NULL DEFAULT 3,
      error_message TEXT,
      created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
      updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
      started_at TIMESTAMPTZ,
      completed_at TIMESTAMPTZ
  );
  CREATE INDEX idx_transcode_jobs_status ON transcode_jobs(status, created_at);

Why PostgreSQL as queue: Already in the stack. SELECT ... FOR UPDATE SKIP LOCKED
is sufficient for expected volume. Migrate to Kafka/RabbitMQ later if needed.

PHASE 3: BACKEND - TRANSCODE SERVICE (WORKER)
================================================================================

Goal: Poll for pending jobs, download raw video, transcode to HLS, upload.

3a. New worker module
    File: backend/transcode/worker/worker.go (new)

    TranscodeWorker struct:
      - db           *pgx.Pool
      - storage      storage.Storage  (existing MinIO interface)
      - config       *config.Config
      - logger       *zap.Logger

    Start(ctx):
      Loop every 5 seconds:
      1. SELECT pending job (FOR UPDATE SKIP LOCKED)
      2. Mark as "processing", set started_at
      3. Download raw file from MinIO to temp dir
      4. FFmpeg transcode to HLS (reuse existing patterns)
      5. Upload HLS segments to MinIO ({vodId}/{quality}/segment-XXXX.ts)
      6. Call livestream service: VOD status="ready", set playbackUrl
      7. Mark job "completed", set completed_at
      8. Delete raw file from MinIO
      9. On error: increment attempts, mark "failed" at max_attempts

3b. Adapt FFmpeg for file input
    File: backend/transcode/transcoder/ffmpeg.go (extend)

    TranscodeFile(ctx, inputPath, outputDir):
      - Same args as existing (libx264, AAC, HLS, multi-quality)
      - Input: -i {inputPath} instead of pipe:0
      - Generate thumbnail (reuse existing logic)
      - Blocking wait for FFmpeg completion

3c. Gateway extension
    File: backend/transcode/gateway/livestream_gateway.go (extend)
    - Add UpdateVODStatus(ctx, vodId, status, playbackUrl)

3d. Wire into main.go
    File: backend/transcode/cmd/main.go
    - Initialize TranscodeWorker alongside RTMP server
    - Start in goroutine, add to graceful shutdown

      Future<ApiResponse<T>> uploadWithProgress<T>(
        String path, {
        required FormData formData,
        T Function(dynamic)? fromJsonT,
        void Function(int sent, int total)? onSendProgress,
      }) async {
        final response = await _dio.post(
          path,
          data: formData,
          options: Options(headers: {'Content-Type': 'multipart/form-data'}),
          onSendProgress: onSendProgress,
        );
        return ApiResponse.fromJson(response.data, fromJsonT: fromJsonT);
      }

    Dio already supports onSendProgress natively - just need to expose it.

5c. Repository method
    File: mobile/lib/features/vod/data/vod_repository.dart

      Future<ApiResponse<Vod>> uploadVod({
        required File videoFile,
        required String title,
        required String description,
        String visibility = 'public',
        void Function(int sent, int total)? onSendProgress,
      }) async {
        final formData = FormData.fromMap({
          'file': await MultipartFile.fromFile(videoFile.path),
          'title': title,
          'description': description,
          'visibility': visibility,
        });
        return _apiClient.uploadWithProgress(
          ApiEndpoints.vodUpload,
          formData: formData,
          fromJsonT: (json) => Vod.fromJson(json),
          onSendProgress: onSendProgress,
        );
      }

5d. Upload progress dialog widget
    File: mobile/lib/features/vod/presentation/upload_progress_dialog.dart (new)

    A persistent dialog that shows upload progress. Uses showDialog (matching
    the existing pattern from conversation_screen.dart and vods_settings_screen.dart).

    Widget structure:
    +-------------------------------------------------+
    |           Uploading Video                        |
    |                                                  |
    |  [============================------]  78%       |
    |                                                  |
    |  45.2 MB / 58.0 MB                              |
    |                                                  |
    |  Status: Uploading...                            |
    |  (or)    Processing on server...                 |
    |  (or)    Upload complete!                        |
    |  (or)    Upload failed. Tap to retry.            |
    |                                                  |
    |              [ Cancel ]                          |
    +-------------------------------------------------+

    Implementation details:
    - Use StatefulWidget inside showDialog
    - FLinearProgress (from Forui) for the progress bar, or LinearProgressIndicator
    - States: uploading -> processing -> complete / failed
    - During "uploading": show real-time bytes sent/total from Dio onSendProgress
    - During "processing": show indeterminate progress (server is transcoding)
    - On "complete": show success icon, auto-dismiss after 2s or tap to view VOD
    - On "failed": show error message + retry button
    - Cancel button: uses CancelToken from Dio to abort the upload
    - Dialog is NOT dismissible by tapping outside (barrierDismissible: false)
    - Format bytes as human-readable (KB/MB/GB)

    Key class:

      class UploadProgressDialog extends StatefulWidget {
        final File videoFile;
        final String title;
        final String description;
        final String visibility;
        const UploadProgressDialog({...});
      }

      class _UploadProgressDialogState extends State<UploadProgressDialog> {
        double _progress = 0.0;         // 0.0 to 1.0
        int _bytesSent = 0;
        int _totalBytes = 0;
        UploadStatus _status = UploadStatus.uploading;
        CancelToken? _cancelToken;
        String? _errorMessage;

        @override
        void initState() {
          super.initState();
          _startUpload();
        }

        Future<void> _startUpload() async {
          _cancelToken = CancelToken();
          try {
            final repo = ... // get from ref or pass in
            final response = await repo.uploadVod(
              videoFile: widget.videoFile,
              title: widget.title,
              description: widget.description,
              visibility: widget.visibility,
              onSendProgress: (sent, total) {
                if (!mounted) return;
                setState(() {
                  _bytesSent = sent;
                  _totalBytes = total;
                  _progress = total > 0 ? sent / total : 0;
                });
              },
            );
            if (!mounted) return;
            setState(() {
              _status = response.success
                ? UploadStatus.processing  // server now transcoding
                : UploadStatus.failed;
            });
          } on DioException catch (e) {
            if (e.type == DioExceptionType.cancel) return;
            setState(() => _status = UploadStatus.failed);
          }
        }

        void _cancel() {
          _cancelToken?.cancel();
          Navigator.of(context).pop();
        }

        // ... build method with progress UI
      }

      enum UploadStatus { uploading, processing, complete, failed }

    Helper - format bytes:

      String formatBytes(int bytes) {
        if (bytes < 1024) return '$bytes B';
        if (bytes < 1048576) return '${(bytes / 1024).toStringAsFixed(1)} KB';
        if (bytes < 1073741824) return '${(bytes / 1048576).toStringAsFixed(1)} MB';
        return '${(bytes / 1073741824).toStringAsFixed(1)} GB';
      }
